Getting Started

This documentation guides you through the process of deploying Prometheus and Grafana monitoring stack on your Kubernetes clusters using our Monitoring Accelerator. The solution supports both Amazon EKS and Google GKE, providing a unified monitoring experience across cloud platforms.

Prerequisites

Before deploying the monitoring stack, ensure you have the following prerequisites in place:

  • An active Amazon EKS or Google GKE cluster
  • kubectl configured to access your cluster
  • Necessary IAM permissions for cluster access
  • Helm 3.x installed on your local machine
  • Minimum 4 vCPUs and 8GB RAM available in your cluster

Deployment Process

The deployment process is streamlined through our web interface. Follow these steps:

  • Select your cluster type (EKS or GKE) from the navigation menu
  • Fill in the required configuration parameters in the form
  • Review your settings
  • Click the Deploy button to start the installation
Note: The deployment typically takes 5-10 minutes to complete. You can monitor the progress in your cluster using: kubectl get pods -n monitoring

Post-Deployment Configuration

After successful deployment, you'll have access to:

  • Prometheus UI at http://prometheus.your-domain
  • Grafana dashboard at http://grafana.your-domain
  • Pre-configured alerting rules
  • Default dashboards for cluster monitoring

The default Grafana login credentials will be displayed after deployment. Make sure to change these credentials immediately for security purposes.
